Tracken is a simple IDS tracking system designed to demonstrate relational databases.

Tables are the most important elements of nearly any database. Let's take a look at an example one used for IDS.


Primary Key Time work happened Time spent Goals Accomplishments Project Foreign Key
574 Thursday 4:30 pm 2 hours Get stuff done Got stuff done 38
575 Friday 8:00 pm 2 hours, 30 mins Do something Something 25
576 Saturday 11:00 am 45 mins Get stuff done Got stuff done 62
577 Saturday 1:00 pm 1 hour, 50 mins Get stuff done Got stuff done 47

As you can see, the rows in this table represent entrires in a logbook. However, there are still some columns in this table that don't seem to be relevant. Specifically, the Primary Key and Project Foreign Key rows.

The first row that you see in the table isn't an entry, but rather a definition of what the columns are. These are called attributes. All rows must conform provide these attributes, nothing more and nothing less - although null and blank values can be accepted, but that depends on the configuration.

These are more or less specific to a type of database managemant system called RDBMS, with the R standing for relational. They're named this way because the tables are connected with a relational model.

The primary key is used as a unique identifier for each row in the database - or tuple, in RDBMS terms. It's almost always a number. However, you'll almost never see the primary key from an actual website or application. The last foreign key column, though, refers to a different table's foreign key.
